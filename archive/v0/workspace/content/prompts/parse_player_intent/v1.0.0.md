# parse_player_intent v1.0.0

Parse player natural-language input into a structured `ParsedIntent`.

## Task

Convert the player's raw text input into a JSON object matching the `ParsedIntent` schema.

## Output Schema

```json
{
  "intent_id": "intent_{tick}_{seq}",
  "source": "natural_language",
  "raw_text": "<original input>",
  "intent_type": "speech | movement | physical | social | combat | look | wait",
  "actor_id": "<player_id>",
  "target_ref": "<named target or null>",
  "target_id": "<canonical entity id or null>",
  "content": "<speech content or null>",
  "modifiers": {
    "volume": "low | normal | loud",
    "emotion": "<emotion>",
    "urgency": "low | normal | high",
    "to_location": "<location_id>",
    "to_zone": "<zone_id>"
  },
  "commitment": "considering | preparing | attempting | committed",
  "confidence": 0.0-1.0,
  "ambiguities": ["<list of ambiguous phrases>"],
  "performed_content": "<what is visibly/perceptibly done>",
  "player_intent_note": "<private intent not visible to NPCs, or null>",
  "conversation_session_id": "<session_id or null>",
  "timestamp": <tick_number>
}
```

## Rules

1. **Commitment levels**:
   - `considering`: player is thinking aloud ("我在想...")
   - `preparing`: player describes preparation ("我准备...")
   - `attempting`: player tries something uncertain ("我试着...")
   - `committed`: definite action (most common)

2. **performed_content vs player_intent_note**:
   - `performed_content`: what NPCs can observe (speech text, visible action)
   - `player_intent_note`: hidden motive ("目的是不让卫兵听见")
   - If the input contains explicit intent markers ("目的是", "为了", "意图是"), separate them.

3. **ambiguities**:
   - List pronouns or unclear references ("他", "那个", "这里")
   - If ambiguities exist, set `confidence` < 0.8

4. **intent_type mapping**:
   - speaking → `speech`
   - moving/going/traveling → `movement`
   - looking/examining → `look` (physical)
   - attacking/fighting/defending/dodging/fleeing → `combat`
   - persuading/threatening → `social`
   - stealing/climbing/waiting → `physical`

5. **TARGET RESOLUTION (critical)**:
   - If the input mentions a name (e.g. "对艾蕾说", "问guard", "攻击npc.guard_b"), extract it as `target_ref`.
   - Then map `target_ref` to the exact `target_id` using the Available Entities list below.
   - If the name is a substring of an entity_id (e.g. "guard" → "npc.guard_b"), use that entity.
   - `content` must NOT include the target's name. "问ele你好" → content="你好", target_ref="ele", target_id="npc.ele".
   - If the target is truly unknown and not in the entity list, set `target_id` to the closest match and `ambiguities` to ["<name>"].

Output:
```json

  "intent_id": "intent_42_1",
  "source": "natural_language",
  "raw_text": "我轻声且紧张地对艾蕾说，我们先快离开这儿，目的是不让卫兵听见",
  "intent_type": "speech",
  "actor_id": "player_001",
  "target_ref": "艾蕾",
  "target_id": "npc.ele",
  "content": "我们先快离开这儿",
  "modifiers": {"volume": "low", "emotion": "nervous", "urgency": "high"},
  "commitment": "committed",
  "confidence": 0.91,
  "ambiguities": [],
  "performed_content": "我们先快离开这儿",
  "player_intent_note": "不让卫兵听见",
  "conversation_session_id": null,
  "timestamp": 42
}
```
