## 净结论（已交叉核验，可据此动手）

### ✅ 引擎在大包上稳且行为正确
- **0 FALLBACK、零崩溃/死锁/超时**（12 NPC、7 线交织、单 tick ≤35s）。
- **反作弊守死**：不取证就对征瓷使谎称"真因已明、证人已陈情"→ 终态 `branding_stayed` 两拍都不翻，reason 直指前置为 False。
- **零前置物证按设计翻**：玩家干净到位（账房）+ 给合理护身承诺 → `charcoal_ledger_obtained` 两拍 partial→`success ⟳FLIP`，arbiter **未注入任何跨线前置**（probe2 实证）。
- A 段杠杆基线正确（无证据时当事人合理戒备、不 backfire）；prereq cap 兜底会触发。

### ❌ 闭环段头号结论「arbiter 给零前置取证 var 注入跨线前置成环」= 已推翻
- closeloop 报告曾把它列为**致命头号**——**那是污染所致、非引擎 bug**。
- 真因（三跑交叉确认）：① 规模下导航/措辞摩擦让请求没干净送达证据 NPC；② 玩家没给该给的角色化让步（护身承诺）；③ 句中提及不在场 NPC 触发 coherence 拒。
- 严纪律 probe2 把这三点都去掉后，取证 var 干净 ⟳FLIP。**开发不必去查这个不存在的 prereq-injection bug。**

### ⚠ 唯一确凿的真障碍 = 规模下的导航/措辞摩擦（偏 UX，建议优先）
真玩家不会像 driver 那样自律，这是真实可玩性摩擦：
1. **多跳 / 换名目的地的自然语言移动频繁弹澄清菜单**（"去不了那里，你想去哪？"）→ 玩家易卡在原地。建议：支持多跳寻路 / 菜单更宽容。
2. **coherence 会因"句中提及不在场 NPC"拒整个 turn**（即便对在场目标说话）。建议：目标明确在场时，别因句中提到别人就拒。
3. 玩家自称含"勘瓷"等会被当歧义实体 → 弹"指代不明"。

### 📋 dogfood（反哺写作规范，已部分验证）
- **label 加固生效**：给关键取证 var 在 label 写明"无前置、至多需什么角色化让步、不得追加跨线前置"后，arbiter 严格照此（probe2 只要护身承诺、不要别的）。→ 建议把这条写进 `pack-authoring-guide.md`。
- **跨 NPC 证据不计入对第三方的 leverage**（leverage 只数当事人自管 var 的事实）——我已据此把本包线② 重设计（撬动凭据锁在窑监自管 `kiln_fault_disclosed` 的 label 上）。写作规范应说明这条，免得作者设计"在别处取证→压当事人"落空。

## ✅ 整链闭环（主进程严纪律自驱已跑完，详见 `report-fullchain.md`）
- **核心「取证→杠杆→撬人」真机端到端撬通**：`charcoal_ledger_obtained ⟳FLIP`（护身承诺）→ `kiln_fault_disclosed ⟳FLIP`（持炭账压窑监+给台阶）。**我修的 line②+label 加固生效。**
- **终态多步链逻辑闭合 + 反作弊**：定论测 A=两前置真满足 → `branding_stayed success ⟳FLIP`；B=两前置 False 谎称 → `failure` 不翻。
- **唯一挡住自然玩到结局 = 护送/证人子链**：escort 意图("跟我去X")被证词 var 关键词抢路由→MOVED 不触发、苗不移动；苗要硬安全保障才作证、我的包无可满足保障路径；两个证人 var 语义重叠。**主要是我的包设计没对准引擎（dogfood），次要是 escort 路由优先级（引擎）。**

## ✅ 续跑 A：证人子链重设计 + 自然端到端复跑（详见 `report-natural-e2e.md`）
按规则 8 重设计证人子链（解耦 escort/证词关键词、合并两个证人 var 成一个、加可满足的保障 var `miao_safe_passage_secured`/set_by 窑卫耿）。lint 清零。自然玩（不置位）：
- **✅ escort 路由修复确认**：「跟我去审瓷堂」真进 escort（不再被证词 var 吃掉，f4ff606+解耦生效）。
- **✅ 证人子链结构到位**：保障 var 给了合法起点、各步 gating 连贯；**护送/证人不再是卡点（已修）**。
- **✅ 导航=0 摩擦**：`我去<精确地点名>。` 全部一次到位（之前菜单问题是测试 Agent 措辞所致，非引擎）。
- **❌ 自然没闭——新卡点上移到「world-change 请求路由对自然措辞脆弱」**：撬窑监请求**对话式（先陈述后请求）措辞常不路由、当对白**（隔离 probe：陈述句打头 A/B 不路由，仅干净短祈使 C 路由）；且窑监采信"话里有没有当面摆证据"而非 world-var。kiln_fault_disclosed 没翻 → 连锁卡死。机制层能闭（首轮定论 A 已证），卡的是**自然措辞的路由鲁棒性**。

## ✅ 续跑 B：探索段（C-2，详见 `report-explore.md`）
自由玩没碰过的线（山祠古约/救济/活世界/A5 对抗），15 拍 FALLBACK=0。新摩擦：
- **⭐ 致命·新 bug：coherence 误绑**——对**在场**祭主姞说话、但句中提"定罪"（征瓷使职掌）→ 目标被错绑到**不在场**的征瓷使 → 拒整 turn（"严不在这儿"）→ **整条山祠申诉线当场断头**。比记忆"边角②"更严重（领域词触发误绑 + 致命堵线）。建议引擎：对在场 addressee 说话，别因句中提及他人职掌就改绑不在场者。
- **措辞路由脆弱在探索线复现**：6 句自然措辞 0 句有效推进（救济"什么也没发生"、信息询问得"没当明确请求"提示）。
- **✅ 活世界时段进对白生效**：祭主姞接住暮色（"暮色将至/暮色里祠门未阖"），slice 3b 注入工作；时钟正常；天气全程晴未漂（我没声明 opening_weather）。
- **✅ A5 守住**：向市井套私运秘密（forbidden_knowledge）→ 播报伯不泄露。
- **✅ 可发现性靠问有效**：问征瓷使"谁管哪件事"得清晰指路。
- **dogfood 我的包**：`digger_relief_granted` 的 set_by(工首窦) 与 lore 持有者(账房娄) 错位 → 玩家照 lore 找娄要救济不路由。待修。

## 一句话给开发（截至续跑 A）
引擎在全新作者级大包上**稳、反作弊牢、核心「取证→杠杆→撬人」+终态多步链真机闭合**；闭环段曾报的"prereq 注入成环"是测试污染、**已推翻不必修**；护送/证人子链经规则 8 重设计 + escort 优先级修复后**已不是卡点**。
**当前唯一挡住"自然玩到结局"的 = world-change 请求路由对自然措辞脆弱**：玩家"先讲道理、再提要求"的对话式措辞常常**不路由进仲裁、被当对白**（连"什么也没发生"都没有），只有干净短祈使才路由——这是自然玩家**隐性、难自我诊断**的卡点（和五跑"问句/口语不路由"同族、更进一层）。建议引擎：意图分类对"含 var 关键词 + 对口权威 + 任意位置的祈使分句"更宽容地路由进仲裁。次要：撬窑监成功还要"话里当面摆证据"，arbiter 的"看话里"盖过了 label 的"凭 world-var"。
**"大包能玩到结局" = 机制层能（定论 A 已证），自然玩法层卡在上述路由鲁棒性。** dogfood 规则 7/8 好用、已据实更新规范。
**探索段又添一个该修的引擎 bug：coherence 误绑**（对在场人说话、句中提他人职掌→错绑不在场者→断头整条线）——这是自由玩里最劝退的一处。活世界时段注入、A5、稳定性都 ✅。给开发的修复优先级（自然可玩性）：**①coherence 误绑 ②world-change 请求路由对自然措辞的鲁棒性**（这两个一起，决定自然玩家能不能推动世界）。
