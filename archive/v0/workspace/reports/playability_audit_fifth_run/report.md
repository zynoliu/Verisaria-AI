## ⭐ 按「伤好玩」排序的摩擦清单（下一轮修复输入）

**1.【明显·维度4/7】玩家"找对人、说对事"但用问句/口语措辞时，请求被静默丢成对白（"什么也没发生"），看不出该怎么说才算数** — 可用性/可发现性
> ⚠ 此条经补充 probe 跑（`report-addendum.md`）**两次纠正**：① 初判"无出口"错——出口存在（`broadcast_retraction_issued`←伊娃、`valley_heat_passes_released`←塔姆辛，玩家都找对了人）；② 再判"冷启动引擎路由缺口"也被 probe 推翻——**冷启动空 ledger 照样能路由**。最终判定如下。

probe 跑（6 拍真机）实证：同样向伊娃/塔姆辛、改用**祈使+命中 var 关键词**的措辞，**2/2 都路由进 Channel-C**（伊娃 `broadcast_retraction_issued → failure`；塔姆辛 `valley_heat_passes_released → partial_success`，还写 ledger + spawn 动态前置）。第五跑 0 路由的真因是玩家措辞卡在两道门（`session.py:1672-1726`）：
- **(a) 问句门** `_looks_like_question`：句末 `?` / "你敢不敢/能不能" 被当讨论、静默丢掉（第五跑对伊娃正是"你敢不敢…撤了？"）。
- **(b) 关键词门**：不含 `request_keywords` 子串、又无既有 ledger 兜底 → 掉成普通对白。
**为何仍伤好玩**：玩家"找对人、说对事"却因措辞偏口语/问句而**零反馈**（只回"什么也没发生"），看不出"换个说法就成"——可发现性差，真诚投入打水漂。**这是可用性问题，不是引擎路由缺口**（路由本身工作正常）。
- 修法（见 addendum）：放宽问句门（命中关键词/对口权威时让位给路由）；纯口语请求至少回"她没把这当正式请求"的可读提示，而非"什么也没发生"。

**2.【明显·维度3/4】当事人苏拉对真诚+给台阶零反应，suspicion 一上来死钉 0.2** — NPC 可信度/选择分量
玩家明确给台阶（"我不想毁你的兰房""可以不写你的名字"），苏拉 suspicion 全程 0.2、trust 0.000 纹丝不动（即便第一轮纯给台阶也 0 位移）。#5 的"真诚降怀疑"在 tidebreak 旁观者上成立，但**当事人这里没触发**。疑似该 NPC 初始 suspicion 偏硬 / "降怀疑"在被追问语境不触发。

**3.【明显·维度2】"环顾四周"刷屏 + "什么也没发生"叠加，节奏闷** — 节奏
几乎每个在场 NPC 每拍都"环顾四周"，叠加说服无果的"什么也没发生"，14 拍读下来很闷、缺"事在动"的反馈。

**4.【轻微·维度7】开局信息过载、不知哪条线有出口** — 可上手
12 个 NPC + 多条压力事件线同时开场，可上手但**不知道哪条线有出口、哪个 NPC 管哪件事**——这与摩擦#1 互相加重：玩家看不出"伊娃管广播撤回、塔姆辛管通行证"，找对了也得不到反馈。压力事件密集刷但与玩家行动无因果连接，像背景刷屏。

## ⭐ 给开发的判断
本跑把"可玩性下一道坎"定位得很准，且**纠正了一个易误判的方向**：valley 线**不是没出口**（`valley_heat_passes_released`/`broadcast_retraction_issued` 都配好了、对口 NPC 也对），问题在**冷启动的权威请求没被路由进 world-change**——玩家找对人、说对事，引擎却只当普通对白。这与二跑 #4「跟进路由」生效形成对照：**#4 在已有 ledger 时能路由，但首次接触（冷启动、无 ledger）路由没覆盖**。
建议三条（按对症）：
1. **【最对症】补冷启动路由**：当玩家**当面向某 world_var 的 set_by 角色持有者**说话、且内容与该 var 的 label/keywords 语义重叠时，即便无既有 ledger 也应路由进 Channel-C（让 arbiter 去判，而非默默丢成对白）。
2. **lint 增"线级出口可达性"提示**：现在 lint 全绿但玩家够不着出口；可提示"某 world_var 的 set_by 角色持有 NPC 是谁、玩家如何能触达"，或在 pack 写作规范加一条"每个出口 var 的持有 NPC 应在某条玩家自然会走的线上"。
3. **F2 覆盖更多说服语境**（很多有质量的说服仍零位移）。
