# 游戏视角模式 — 远期规划（含第二人称"顾问"模式）

> 记录于 2026-06-01。**远期方向**，非当前任务。等核心玩法（PLAY-3 三通道）+ UI 稳定后再立项。
> 起因：用户提出一个差异化点子——市面少见的**第二人称**玩法。

---

## 0. 核心点子（第二人称 / "顾问"模式）

玩家**不是**主角（第一人称），也**不是**上帝视角的操控者（第三人称），而是某个角色的
**脑内音 / 顾问 / 同伴**（类比原神的派蒙）。**游玩方式 = 与角色对话、给建议**；角色**有自主
意识**，自己决定听不听、怎么执行。

示例：
```
角色：看起来队长并不愿意打开城门，我们该怎么办呀？
玩家：要不你试试拿出委托令，向他们坦白你是陛下委托而来暗中处理难民事物的。
角色：有道理，我试试吧。
   → 角色去跟队长周旋 → 队长裁决 → 世界回应
```

---

## 1. 为什么它**特别契合这套引擎**

第二人称的核心循环 = **玩家给建议 → 自主角色决定听不听 → 角色行动 → 世界回应**。

这正是 **Channel C 触发**（"说服权威 NPC → 他基于关系+论据+职责自己裁决 → 行动 → 世界变"）
的同构循环，只是**从偶尔变成核心、并加一层间接**：
```
玩家建议主角 → 主角基于【对玩家的信任 + 建议是否合理 + 自己的性格】决定听不听
            → 主角去说服队长 → 队长裁决 → 世界变
```

**现有引擎可复用 ~80%**：自主 NPC（主角就是其一）、关系系统（advisor↔主角 = "他多信你的建议"
这一核心数值）、说服/裁决（arbiter 判断主角是否采纳）、角色用自己嗓音流式回应、世界对行动的反应。
**难的部分（自主角色 + 关系闸门的服从）已做完**；剩下主要是**控制模型反转**，不是新核心机制。

---

## 2. 它顺手解决的两个老问题

1. **A5 视角难题**（"可见真相该谁看"）：作为角色的脑内音，玩家**理所当然共享角色的认知**——
   你看到的世界就是角色所知的世界。信息不对称问题**自动消解**（你就是角色的那部分心智）。
2. **意图被解析错**（最担心的脆弱点）：第二人称里玩家输入是**给角色的建议**，角色像人一样
   **领会**它，不懂还能**在剧情里反问**（"你是说让我亮出委托令？"）。脆弱的意图解析层被
   "角色作为人来领会"替代——误解成了自然对话节拍，而非机械失败。

---

## 3. 架构（用户拍板 + 我的具体化）

**两模式共享同一核心引擎，只在控制/视角层不同；模式由剧本声明、锁定、不可中途切换。**

- **共享核心（mode-agnostic，绝不让模式逻辑渗入）**：自主 agent、信念/记忆、关系、arbiter、
  世界状态、campaign driver。
- **薄控制层（按模式分流）**：
  - `protagonist` 模式（当前 第一/三人称）：玩家=世界里的实体；输入=世界动作；按感知过滤看世界。
  - `advisor` 模式（第二人称）：玩家=顾问，在世界里**无身体**；输入=**给主角的建议**；
    通过**主角的认知**看世界；**主角**是世界里的行动者。
- **关键抽象**：**"焦点实体" + "玩家角色"**
  - protagonist：焦点=玩家实体，玩家=actor。
  - advisor：焦点=被建议的主角，玩家=advisor；"顾问↔主角"是条特殊关系（建议坑了他 → 他不信你了，
    本身是一条后果链）。
- **pack 字段**（建议）：`play_mode: "protagonist" | "advisor"`，缺省 `protagonist`（现有剧本向后兼容）。
  advisor 剧本额外声明：主角是谁（焦点实体）、开场情境、主角性格/目标。
- **锁定的好处**：引擎加载时分支一次，运行时各走各的、稳定；第二人称作为**独立适配器**叠加，
  **不动摇现有第一人称路径**（加法，非改造，低风险）。

> 与 [ui-architecture-direction.md](ui-architecture-direction.md) 衔接：`play_mode` 应是结构化
> 视图模型/控制接口的一个参数；UI 渲染"玩家↔主角(建议) + 主角↔世界(行动)"两层。

---

## 4. 真正的难点（make-or-break）：自主性的度

角色能拒绝建议。太自主 → 玩家挫败（"我让他做 X 他偏不"）；太听话 → 提线木偶（背叛前提）。
甜点区：**好建议（尤其信任建立后）会采纳，坏/突兀建议会合理质疑，永远用角色自己的方式执行**。
调节它的正是 **advisor↔主角的关系**（开局半信半疑 → 你出好主意、共担后果 → 越来越信你）。
**只能靠小原型"手感测试"调出来。**

---

## 5. 建议的第一步（不要一上来重构）

用 frostgate，选**一个主角**（如一个奉命暗中处理难民的低阶军官），开场他对玩家（脑内音）说
"队长不肯开门，怎么办呀"，玩家给建议，测：
1. 角色领会建议自然吗？
2. 采纳/质疑的判断舒服吗？
3. 看着他用你的主意去跟队长周旋、世界真的变了——爽不爽？

**就一个场景**，把"度"调舒服，再决定要不要把它做成主玩法。原型复用现有引擎 ~80%，主要写
"主角的建议-决策循环"这一层 + `advisor` 控制适配器。
