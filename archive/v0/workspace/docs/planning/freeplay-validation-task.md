# 测试任务：v0.3.0 自由试玩验证（移动已修，找 P2c 信号）

> 给测试 Agent 的任务简报。承接动态世界模型里程碑 `v0.3.0`
> （`PASS_P1_P2B_PROCESS_CLOSURE`，详见 `docs/design/dynamic-world-model.md`）。

## 背景

六轮迭代后，P1（动态前置变量）+ P2b（线下流程成熟）已真机闭环。这轮**不再是定向验证某个机制**，
而是**自由试玩**——而且**移动已修**：coherence 现在允许**多跳可达**移动（`commit ed67d49`），
"去某地"/"去找某 NPC"只要图上有路径就能一句话走到，**不必再用 `/inject` 挪位**。

## 怎么跑

- 真机 + `--log`，**尽量全程用自然语言**（移动也用自然语言，先别用 `/inject`，看它顺不顺）：
  ```bash
  PYTHONPATH=src python -m verisaria run fixtures/content_packs/skyglass_memory_inquest.json \
    --llm minimax --log reports/<新目录>/run.log
  ```
- 走一条你觉得自然的调查/谈判线（不必非得复刻之前那条）。目标是体验真实可玩性 + 暴露下一层问题。

## 关注点（逐条回答）

1. **移动是否顺了**：自然语言"去 X""去找某 NPC"现在能多跳直达吗？还有没有移动相关的摩擦
   （路径解析失败、状态栏 zone 显示成 location id 等）？贴例子。
2. **闭环体验是否自然**：移动顺畅后，P1+P2b 的链读起来像连续谈判吗？流程启动时 NPC 是否**明确说出
   "已经去办、需要点时间"**（这是上一版刚补的可见对白）？有没有新暴露的问题。
3. **是否撞到 P2c 信号（关键）**：有没有遇到一个前置，**必须把某个【现有】NPC 带到某地、或当面见证/
   出示物证**才能满足——而不是"提交线下流程等成熟"（那是 P2b 已覆盖的）？
   - 若有：**具体记下**——哪个 NPC、要带到哪、要见证什么、你尝试了什么自然语言、引擎怎么反应
     （是无路可走、还是退化成普通对白）。这决定 P2c 要不要做、做多厚。
   - 若没撞到：说明这条线 P1+P2b 就够，P2c 可暂缓。
4. **反作弊抽查一次**（伪造某前置已完成，对应变量仍 False 时不应翻终态）。剔除 `⚠FALLBACK` tick。

## 报告请包含

- 整体可玩性观感（移动顺不顺、闭环自不自然）。
- **P2c 信号：有/无 + 具体例子**（这是本轮最重要的产出）。
- 任何新暴露的问题（移动、对白、解析、状态显示…）。
- 新 `run.log` + transcript 放 `reports/<新目录>/`。

## 一句话目标

用顺畅的自由试玩确认 v0.3.0 的真实手感，并**用真机判断 P2c（现场/召集）到底要不要做、做多厚**——
而不是凭空建。
