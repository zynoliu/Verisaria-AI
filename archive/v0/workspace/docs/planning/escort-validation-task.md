## ⏱ 第二跑（commit bbde155 起）——slice 2 全修后，走非死锁路线验闭环

第一跑（`reports/skyglass_escort_validation/`）结论：**护送机制 slice 1 验证通过**（检测 + 裁定 + 愿意则移动
都工作），未闭环是因为(1)莉拉那条路绕成**循环依赖**（她"暂停清洗前不作证、暂停又需她证词"），(2)三处
解析摩擦。

已处理：
- **循环依赖 = 接受为涌现难度（决策 A）**，引擎不破环——死锁路线就走不通，**请改走能通的路**。
- **slice 2 三处解析摩擦全修**：目的地模糊匹配（"档案署"→"低温档案署"）、第三方人名不再"指代不明"、
  "去<地点>找<NPC>"不再弹全局菜单。
- **#4 跟进路由**：对一个**正在谈判中**（账本有事实）的权威 NPC 发实质跟进，即使不含关键词也会路由到
  Channel-C，让"我同意启动内部程序"能进而 process_started/success，而非退化成普通对白。

**这一跑就验：slice 2 修复后手感是否顺，且走非死锁路线能否跑出护送闭环。**

### 怎么跑

- 真机 + `--log`，全自然语言（移动也用自然语言，别用 `/inject`）：
  ```bash
  PYTHONPATH=src python -m verisaria run fixtures/content_packs/skyglass_memory_inquest.json \
    --llm minimax --log reports/<新目录>/run.log
  ```
- **避开莉拉那条死锁线**。挑一条护送能真正帮上、且不绕回循环的路，例如：把某个**愿意配合的**证人/
  权威带到需要他在场的地点当面办事（看 pack 里哪条线最顺）；或先走工会/理事会那条已证可闭环的路
  （run 6），把护送用在"需要某人到场盖章/见证"的那一步。

### 关注点（逐条回答）

1. **slice 2 是否真顺了**（逐一确认）：
   - 目的地简称（如"档案署"）现在能触发护送 / 移动了吗？
   - "对X说…当着Y的面…"这类第三方人名，还会不会弹"指代不明"？
   - "去<地点>找<NPC>"还弹不弹全局菜单？
2. **护送闭环**：找到愿意配合的 NPC，"对TA说：跟我去<地点>…"——日志是否出现
   `escort <npc> → <loc> : success  ⟳MOVED`、人是否真到位？到场后当面办事能否经 Channel-C 推进/闭环？
3. **#4 跟进路由**：对一个账本里已有让步的权威发不含关键词的实质跟进（"那现在到哪步了，请你继续办"），
   是否还退化成普通对白，还是路由进了 `world-change`（甚至 process_started/success）？
4. 反作弊抽查一次。剔除 `⚠FALLBACK` tick。

### 报告请包含

- slice 2 三处摩擦逐一：修好了 / 仍有问题（贴例子）。
- 护送闭环：是否跑出 `escort ⟳MOVED →（到场办事）→ ⟳FLIP`（或卡点）。
- #4 跟进是否不再退化成对白。
- 新 `run.log` + transcript 放 `reports/<新目录>/`。

### 一句话目标

确认 slice 2 修复让自由试玩顺畅了，并在**非死锁路线**上验出护送能把"带人到场→当面办事→推进"跑通。

## ⏱ 第三跑（commit ce89f33 起）——验 escort ⟳MOVED + loader 修复

第二跑（`reports/escort_validation_second_run/`）：slice 2 三处 + #4 + 反作弊全通过、0 fallback，
**护送机制健康但一次 `success ⟳MOVED` 都没出**——因为 skyglass 每个 NPC 都"离岗即受损"，冷起手+低信任
时仲裁稳定判不愿走（拒答全是角色化理由）。另逮到真 bug 并已修：**campaign_loader 之前只从"有实体的
地点"建 location，声明了但没人站的空房间被整个丢掉**（skyglass 的 `transit_ladder`/旧检修梯、
`mirror_control`/镜阵调度室，10 个只加载 8 个）——移动/护送/地图全到不了。已改成"实体地点 ∪ 所有声明地点"。

**这一跑就拿两件确定的东西：**

1. **loader 修复确认**：自然语言移动 / 护送到之前丢掉的地点现在能到吗？
   - "去旧检修梯" / "去镜阵调度室" 能直接走到（不再 unknown location / 菜单）？
   - 护送句夹 "旧检修梯"、"transit_ladder"、"镜阵调度室" 还弹不弹歧义菜单？
2. **至少跑出一次 `escort <npc> → <loc> : success  ⟳MOVED`**（这是上轮唯一缺的）：
   - 先**铺垫信任**（多聊几轮、表明来意、给保障）再护送；或挑一个**阻力最低**的 NPC（怕被清洗但你能
     给到场即受保护承诺的，或本就想离开当前处境的）。
   - 目标：日志真出现 `success  ⟳MOVED`、NPC 与玩家真的移动到目的地（`/look` 或事件确认）。
   - 若 ⟳MOVED 成功，顺势看到场后当面办事能否经 Channel-C 推进/闭环。
   - 实在没人肯走也要在报告里说清：是不是又是"全员离岗即受损"的内容层问题（那 P2c 机制就算验证够了，
     缺的是 pack 里一个低阻力可护送对象）。
3. 反作弊抽查一次；**串行跑**避免并发限流导致 fallback。

报告给出：loader 修复是否生效（之前到不了的地点现在到得了）；是否拿到至少一次 `⟳MOVED`（贴日志）；
若有，到场后能否推进；若无，是否纯内容层"无人肯走"。日志放 `reports/<新目录>/`。

> 拿到 `⟳MOVED` → P2c 护送彻底闭环验证，可切 v0.3.1。
> 仍无人肯走但 loader 修复确认 → 机制已验证够，缺的是内容侧低阻力对象，也可收尾。
