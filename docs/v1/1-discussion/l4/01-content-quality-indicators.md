# Verisaria v1 内容质量指标体系（L4）

**状态**：初始草案（L4 Round 1–5 讨论后）
**日期**：2026-07-04
**版本**：v0.1

---

## 1. 指标分层

### 1.1 Must（强制）—— 3 项

| 指标 | 描述 | 验证方式 |
|------|------|----------|
| Routeability | 所有可达位置必须有明确路径 | 静态分析 + 动态探索 |
| Exit Reachability | 所有出口必须可被玩家发现并使用 | 探索测试 |
| Natural Language Discoverability | 关键信息必须能通过自然语言描述找到 | LLM 辅助验证 |

### 1.2 Recommended（强烈建议）—— 4 项

| 指标 | 描述 |
|------|------|
| NPC 职掌 / 第三方人名清晰度 | NPC 职责和人名应清晰可辨 |
| 状态收敛一致性 | 世界状态在操作后应收敛到一致状态 |
| Faction-gated truth verifiability | 阵营 gated 知识应可被对应阵营验证 |
| Advisor-mode discoverability | Advisor 模式下关键信息应可发现 |

### 1.3 Optional（可选）—— 1 项

| 指标 | 描述 |
|------|------|
| 覆盖率 / 深度 | 内容包的场景覆盖深度 |

---

## 2. 强制 playability proof 门禁

- 内容包发布前必须通过验证
- 可配置 warn / block 模式
- 支持作者自定义验证规则

---

## 3. 与 LLM 内容生成的绑定

- 所有 LLM 生成/修改的内容必须经过指标验证
- 验证失败触发修复循环（最多 3 次）
- 最终必须人工确认后才能发布
